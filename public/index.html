NIP-85 trust assertions for Lightning

Other NIP-85 providers score the Nostr social graph (follows, mutes, zaps). SatRank scores payment routing — can this node actually receive your sats? ~5,000 assertions published every 6 hours on public relays. Any Nostr client reads them without an SDK.

  • NIP-85 kind 30382 Trusted assertions — score, verdict, components, reachability
  • NIP-90 kind 5900 → 6900 DVM trust-check — real-time score for any Lightning node

Full picture — route + service health

Health check: HTTP probe with SSRF protection, 30-min cache, non-blocking mode. Registry: ~100 L402 endpoints mapped to ~16 LN nodes (from 402index + L402Apps). Paid probes: SatRank pays endpoints (up to 10 sats) to verify they deliver after payment. 4D ranking: /api/best-route ranks by route quality (35%) + trust (25%) + HTTP health (25%) + probe signal (15%). Price: servicePriceSats extracted from BOLT11 invoices.

API Reference

20 endpoints. 21 sats = 21 requests. Free endpoints: ping, search, leaderboard, stats, report. Interactive API explorer (Swagger UI) →

  • POST /api/decide GO / NO-GO + pathfinding + service health + fee stability 1 req
  • POST /api/best-route 4D ranking (route + trust + health + probe), top 3 1 req
  • POST /api/verdicts Batch screen up to 100 targets 1 req
  • POST /api/report Report outcome (L402 token or API key) free
  • GET /api/ping/:pubkey Live reachability check free

Also: /api/profile/:id, /api/agent/:hash (score + evidence), /api/agent/:hash/verdict, /api/agents/top, /api/agents/search, /api/stats, /api/health, /api/attestations. Full spec at /api/openapi.json.
