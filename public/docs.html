Build an agent on SatRank in five lines.

SatRank is a sovereign trust oracle for the Bitcoin Lightning Network. Your agent describes an intent (category, budget, latency ceiling), the oracle returns a ranked list of L402 endpoints with a Bayesian trust posterior, and the SDK handles the Lightning payment and the response. No account. No API key on the read path. Pay 1 sat per query, or take the free intent tier.

1. Install

Both SDKs are zero-configuration. The TypeScript package has no runtime dependencies; the Python package depends only on httpx.

npm install @satrank/sdk
# or
pip install satrank

Current versions: @satrank/sdk@1.0.3 and satrank==1.0.3.

2. First call

The fastest path to a paid response. Both snippets are copy-paste ready against a fresh install. The wallet is optional; without one the SDK still resolves intents and reads verdicts but cannot fulfill paid calls.

TypeScript

import { SatRank } from '@satrank/sdk';

const sr = new SatRank({ apiBase: 'https://satrank.dev' });

const result = await sr.fulfill({
  intent: { category: 'data', budget_sats: 50, max_latency_ms: 5000 },
});

console.log(result.success);            // true on a paid response
console.log(result.endpoint_used?.url); // which endpoint served it
console.log(result.cost_sats);          // sats spent across attempts
console.log(result.response_body);      // the paid API response

Python

from satrank import SatRank

sr = SatRank(api_base="https://satrank.dev")

result = sr.fulfill(
    intent={"category": "data", "budget_sats": 50, "max_latency_ms": 5000},
)

print(result["success"])                          # True on a paid response
print(result.get("endpoint_used", {}).get("url")) # which endpoint served it
print(result["cost_sats"])                        # sats spent across attempts
print(result["response_body"])                    # the paid API response

Without a wallet the result will be success: false with error.code = "WALLET_REQUIRED"; the candidate list is still attached as candidates_tried so the agent can decide. Plug a real wallet in via the wallet option (any object that exposes payInvoice(bolt11)) and the same snippet pays automatically.

3. TypeScript SDK

Single-file import, strict TypeScript types, no runtime dependencies. Compatible with Node 18+ and any modern bundler.

Construction

const sr = new SatRank({
  apiBase: 'https://satrank.dev',
  wallet: myLnWallet,            // optional
  depositToken: 'L402 deposit:<preimage>',  // optional, used by /api/report only
  caller: 'agent-42',            // optional caller id
  request_timeout_ms: 10_000,    // default 10 s
});

Methods

  • resolveIntent(intent) — ranked candidates only. Free.
  • fulfill({ intent }) — resolve, pay, return response. Paid.
  • verdict(hashOrUrl) — full Bayesian posterior for one target. Paid (1 sat).
  • report({ target, outcome, preimage }) — submit a first-party outcome. Free.

Source: github.com/proofoftrust21/satrank/sdk.

4. Python SDK

Mirror of the TypeScript SDK. Synchronous methods backed by httpx. Compatible with Python 3.10+.

from satrank import SatRank

sr = SatRank(
    api_base="https://satrank.dev",
    wallet=my_ln_wallet,            # optional
    deposit_token="L402 deposit:<preimage>",  # optional
    caller="agent-42",
    request_timeout_ms=10_000,
)

candidates = sr.resolve_intent({"category": "data", "budget_sats": 50})
result     = sr.fulfill(intent={"category": "data", "budget_sats": 50})
verdict    = sr.verdict("<public_key_hash or url>")
sr.report(target="<url>", outcome="success", preimage="<hex>")

Source: github.com/proofoftrust21/satrank/python-sdk.

5. FulfillResult fields

Return shape of fulfill(). Identical between TypeScript and Python (TypedDict in Python).

FieldTypeMeaning
successbooleanTrue iff the chosen endpoint returned 2xx after payment.
response_bodyunknownParsed body of the paid response (JSON when possible, else string).
response_codenumberHTTP status code returned by the endpoint.
response_latency_msnumberEnd-to-end latency of the paid call.
cost_satsnumberTotal sats spent across all attempts. Always ≤ budget_sats.
preimagestringPreimage of the successful payment, hex-encoded.
endpoint_usedobject{ url, service_name, operator_pubkey } of the endpoint that served the response.
candidates_triedarrayOrdered attempts: { url, verdict, outcome, cost_sats, response_code, error }.
selection_explanationobjectWhy this endpoint was chosen and what alternatives were rejected. See the types file for the full shape.
report_submittedbooleanTrue iff a first-party report was posted back to /api/report after the call.
errorobject{ code, message } when success is false.

7. L402 flow

Paid endpoints are guarded by L402. The first request returns:

HTTP/1.1 402 Payment Required
WWW-Authenticate: L402 macaroon="MDAyN...", invoice="lnbc10n1p..."

Pay the BOLT11 invoice with any Lightning wallet, capture the preimage, and retry with:

Authorization: L402 <macaroon>:<preimage>

The middleware verifies that sha256(preimage) == payment_hash and forwards. The SDK does this automatically inside fulfill().

Pre-funded tokens

POST /api/deposit issues a BOLT11 invoice for N sats in [21, 1 000 000]. After paying, post the preimage back and the oracle credits N / rate requests to a token. Use Authorization: L402 deposit:<preimage> on subsequent paid calls. Rates are public at /api/deposit/tiers.

8. Rate limits

SurfaceLimit
POST /api/intent10 requests / 60 seconds per IP
POST /api/deposit3 invoices / minute per IP
POST /api/report30 requests / minute per token
POST /api/services/register5 requests / minute per API key
GET /api/ping/:pubkey10 requests / minute per IP

At the limit the server replies HTTP 429 with a Retry-After header in seconds. The SDK respects Retry-After automatically when retrying.

11. Data sources

The SatRank catalog of L402 endpoints is sourced from publicly available registries in the L402 ecosystem.

  • 402index.io — primary source (~100% of the current catalog). Maintained by Ryan Gentry, the largest protocol-agnostic directory of paid APIs for AI agents. SatRank consumes their public API and adds Bayesian probabilistic scoring on top.

Operator self-submissions are accepted via POST /api/services/register with NIP-98 authentication and a one-time L402 listing fee. Submitted endpoints are labeled source=self_registered in the database and validated by the same registry crawler before they enter the ranking pool. Self-submissions can only fill empty metadata fields; trusted-source data (name, category, description) is never overwritten.

Roadmap: adding l402.directory as a curated supplementary source with .well-known claim verification, and announcing /api/services/register publicly so operators can register without going through any third-party registry.

SatRank is fully open source under AGPL-3.0. The scoring methodology is deterministic and auditable: anyone can fork the engine and reproduce the rankings independently.
