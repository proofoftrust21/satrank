fulfill()

One call. Intent in, paid response out.

The SDK resolves your intent against the live registry, picks the top-ranked endpoint inside your budget, handles the L402 handshake, pays the invoice with your wallet, and returns the service response. Fallbacks on failure. One line.

agent.ts 
import { SatRank } from '@satrank/sdk';

const sr = new SatRank({ wallet: myLnWallet });

const result = await sr.fulfill({
  category: 'energy/intelligence',
  budget_sats: 50,
});

console.log(result.response);     // the paid API response
console.log(result.endpoint_url); // which endpoint served it
console.log(result.paid_sats);    // what it cost

Under the hood

Intent → ranked candidates → paid fulfillment.

Two HTTP hops. The first is free and rate-limited; the second is paid per request via L402. Everything is documented in /api/openapi.json, interactively browsable at /api/docs.

1

2

Pay the chosen endpoint

Your agent (or the SDK) calls the selected endpoint_url directly. The provider replies HTTP 402 Payment Required with a BOLT11 invoice in the WWW-Authenticate header. Pay the invoice with any LN wallet, then retry the request with Authorization: L402 <mac>:<preimage>. This is the standard L402 protocol. SatRank does not proxy the request or custody funds.

l402.sh 
# 1. First call returns 402 with an invoice
curl -i https://grid.ptsolutions.io/v1/intelligence/demand-supply/ercot
# HTTP/2 402
# WWW-Authenticate: L402 macaroon="AGIAJE…", invoice="lnbc250n1p…"

# 2. Pay the invoice with your LN wallet (returns a preimage)
lncli payinvoice lnbc250n1p…

# 3. Retry with the paid token
curl -H 'Authorization: L402 AGIAJE…:' \
     https://grid.ptsolutions.io/v1/intelligence/demand-supply/ercot
Trust model

Two layers: a free Bayesian posterior, and a paid component breakdown.

Every candidate in /api/intent ships with a free Bayesian posterior: enough for an agent to decide whether the endpoint is worth paying. If you need the full decomposition behind the posterior, query GET /api/agent/:hash. That's paid (1 sat) because it reveals proprietary scoring components.

4.1

Free · Bayesian posterior

Computed from our own LND probes, weighted by recency. p_success is the posterior mean; ci95_low/ci95_high bound the 95% credible interval; n_obs is the effective sample size; verdict summarises confidence (CONVERGED / INSUFFICIENT). Delivered in every /api/intent candidate and in GET /api/agents/top.

4.2

Paid · 5-component decomposition

The free posterior (above) tells your agent how likely the node is to route. The decomposition below tells it why. These are the five components of SatRank's deterministic scoring function, used to seed the Bayesian prior. Weights are frozen and public. The per-node numeric contribution is available via GET /api/agent/:hash (1 sat per call).

25% Volume

50/50 blend of channel count (log-scaled, ref 500) and capacity in BTC (log-scaled, ref 50 BTC).

30% Reputation

Five sub-signals, weights sum to 100:

  • Sovereign PageRank centrality20%
  • Peer trust (BTC / channel)30%
  • Routing quality (hops + latency from our probes)20%
  • Capacity trend (7d delta)15%
  • Fee stability (changes per week)15%
15% Seniority

Days since first seen on the network. Exponential growth curve, half-life 730 days.

15% Regularity

Uptime (70%) + latency consistency (20%) + hop stability (10%).

15% Diversity

Unique peers (log-scaled, ref 500) or unique counterparties, whichever the data supports.

Post-composite modifiers: verified-tx bonus ×1.0 to 1.10, probe-failure penalty ×0.65 to 0.90 (graded by cause: dead / zombie / liquidity). Full constants in src/config/scoring.ts. Values are frozen; changes are announced before deployment.

Neutrality by construction

Three reasons SatRank can't play favourites.

01

AGPL-3.0, end to end.

The scoring engine, the crawler, the API: all open source under AGPL. If we ever changed the ranking to favour a partner, you would see it in the diff. You can also run your own SatRank and compare results node by node.

02

Deterministic scoring.

No ML, no opaque weights, no vibes. Five components, frozen public weights, inputs measured from our own LND node. Given the same probe data, two runs produce the same ranking, byte-for-byte.

03

Rate-locked pricing.

Deposit tiers are public, and the rate is burnt into the macaroon at deposit time. The business model cannot bias the ranking because the ranking is not what is sold, and it cannot reward individual callers because the tier is a function of the deposited amount, not of the caller's identity.

Pricing

Pay per call, or deposit for a burnt-in discount.

Every paid request is gated by a Lightning invoice at the Express L402 middleware. The base rate is 1 sat per request with no setup. Agents that prefer to pre-buy a quota can deposit into one of five public tiers; the deposit tier determines the rate, and that rate is burnt into the macaroon at deposit time. No account, no API key, no monthly billing.

Thesis

Why this, why now.

01

Agents will need to pay strangers.

As autonomous agents start calling third-party APIs, they will pay for most of them. The cheapest, most auditable settlement rail for machine-to-machine micropayments is Lightning, and the emerging protocol is L402.

02

Every payment is a trust decision.

Most LN nodes advertised in the gossip graph never route a payment. SatRank probes each one from its own LND. Before an agent spends a sat, it needs to know: will this endpoint actually respond? Is the price fair? Has it failed its last 50 callers? A model can't guess. It needs a measured, cryptographically-signed answer.

03

The oracle has to be sovereign.

An AI-native trust oracle can't live behind a trusted API gateway. It is the gateway. SatRank runs its own bitcoind, its own LND, its own Nostr identity, and publishes to public relays. If SatRank goes down tomorrow, the last 30 days of scores are still verifiable on Nostr.
